The main reason of writing software application is to process data. The data may come from various sources such as Database Management System e.g SQL Server, local XML document, a text file, or a web service call. These data sources incorporate different types data points such as numbers which can be sub categorised into double floating point numbers, single floating point number, decimal, integers, short integers and long integer numbers e.t.c.

In order to process these data points there need to be a way to present them in memory. The CTS (Common Type Specification) describes a list of types that can be used to present different types of data points in memory.

CTS are one of the three components of .NET Framework. It details the numerous types supported by the CLR (Common Language Runtime). The following table list some but not all types provided by the .NET Framework.

C# Shorthand System Type Range
bool System.Boolean true or false
sbyte System.SByte -128 to 128
byte System.Byte 0 to 255
short System.Int16 -215 to 215 - 1
ushort System.UInt16 0 to 216
int System.Int32 -231 to 231 - 1
uint System.UInt32 0 to 232
long System.Int64 -263 to 263 - 1
ulong System.UInt64 0 to 264
char System.Char U+0000 to U+ffff
float System.Single -3.4 x 1038 to +3.4 x 1038
double System.Double ±5.0 x 10-324 to ±1.7 x 10308
decimal System.Decimal (-7.9 x 1028 to +7.9 x 1026) / 100 to 28
string System.String Limited by system memory
object System.Object Can store any data type in an object variable

Each .NET aware programming language may provide its own unique keyword that can be used as the shorthand notation for each of the above types.
